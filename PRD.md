# Gridlock Garage PRD

## 1. 제품 개요

**제품명:** Gridlock Garage / 그리드락 개러지

**장르:** 3D 슬라이딩 퍼즐 보드게임

**플랫폼:** Web

**핵심 목표:** 독자적인 브랜드, 비주얼, 스테이지 데이터를 가진 3D 차량 퍼즐 게임을 만든다. 사용자는 6x6 차고 보드에서 차량 블록을 앞뒤로 움직여 목표 차량을 출구까지 보내고, 스테이지별 기록 경쟁을 한다.

## 2. 지식재산권 회피 원칙

- 기존 상용 보드게임 명칭, 로고, 패키지 디자인, 차량 조형, 색 구성, 문제집 배치를 사용하지 않는다.
- 게임명은 `Gridlock Garage`를 사용한다.
- 목표 차량, 일반 차량, 보드판의 외형은 독자적인 차고/정비소 테마로 제작한다.
- 스테이지 1-100은 직접 설계하거나 자체 생성 알고리즘으로 만든다.
- 마케팅 문구에서 기존 제품명을 비교 대상으로 사용하지 않는다.

## 3. 핵심 기능

### 3.1 스테이지

- 총 100개 스테이지를 제공한다.
- 스테이지 번호가 높아질수록 차량 수, 방해 차량 수, 필요한 이동 수가 증가한다.
- 각 스테이지는 6x6 보드, 출구 위치, 차량 목록, 목표 차량 정보를 가진다.
- 플레이어는 이전 스테이지 클리어 후 다음 스테이지로 이동할 수 있다.
- MVP에서는 모든 스테이지 선택을 허용하고, 정식 버전에서 순차 해금 옵션을 적용한다.

### 3.2 플레이

- 차량은 배치 방향으로만 이동한다.
- 가로 차량은 좌우, 세로 차량은 상하로 이동한다.
- 다른 차량과 겹치거나 보드 경계를 넘어갈 수 없다.
- 목표 차량이 오른쪽 출구로 빠져나가면 클리어된다.
- 경과 시간과 이동 횟수를 기록한다.
- 되돌리기, 재시작, 다음 스테이지 기능을 제공한다.

### 3.3 리더보드

- 스테이지별 1위부터 50위까지 표시한다.
- 정렬 기준은 이동 횟수 오름차순, 시간 오름차순, 기록 일시 오름차순이다.
- 사용자의 스테이지별 최고 기록만 리더보드에 반영한다.
- MVP에서는 로컬 저장소 기반 리더보드로 시작하고, 서버 연동 시 계정 기반으로 전환한다.

### 3.4 3D 보드게임 경험

- 6x6 홈이 파인 보드판을 3D로 구현한다.
- 차량은 둥근 모서리, 바퀴, 유리창, 그림자를 가진 장난감 자동차 스타일로 표현한다.
- 카메라는 약간 기울어진 탑다운 시점을 기본값으로 사용한다.
- 차량 선택, 이동 가능 칸, 출구를 시각적으로 강조한다.
- 드래그 조작으로 실제 보드게임 말판을 미는 느낌을 만든다.

## 4. 화면 구성

### 4.1 게임 화면

- 상단: 제품명, 현재 스테이지, 난이도, 경과 시간, 이동 횟수
- 중앙: 3D 보드
- 좌측: 스테이지 선택
- 우측: 스테이지별 리더보드
- 하단: 되돌리기, 재시작, 다음 스테이지

### 4.2 클리어 화면

- 클리어 시간
- 이동 횟수
- 개인 최고 기록 여부
- 현재 스테이지 순위
- 다시 하기
- 다음 스테이지

## 5. MVP 범위

- 정적 웹 앱
- 3D 보드 렌더링
- 100개 스테이지 데이터 생성
- 차량 드래그 이동
- 클리어 판정
- 시간/이동 횟수 측정
- 로컬 스토리지 리더보드
- 스테이지별 상위 50위 표시

## 6. 서버 확장 범위

- 사용자 계정
- 서버 검증용 이동 로그
- 기록 제출 API
- 스테이지별 리더보드 API
- 비정상 기록 필터링
- 시즌 랭킹과 친구 랭킹

## 7. 성공 지표

- 스테이지 1 클리어율
- 스테이지 10 도달률
- 스테이지 100 클리어 유저 수
- 평균 재도전 횟수
- 리더보드 조회율
- 기록 갱신률
- 일간 활성 사용자 수
